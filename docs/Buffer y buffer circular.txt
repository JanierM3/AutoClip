¿QUE ES UN BUFFER?

Un buffer es un espacio en memoria (RAM), en el que se almacenan datos de manera temporal,
normalmente para un único uso. Básicamente es como una "zona de espera" donde los datos
seguardan antes de ser procesados o transferidos a otro lugar.

Los buffers funcionan con un sistema de cola (QUEUE) FIFO (First In, First Out),
esto quiere decir: primero en entrar, primero en salir.

Ejemplo visual:
┌─────────────────────────────────────────┐
│  Buffer (cola FIFO)                    │
├─────────────────────────────────────────┤
│  Entrada → [A] [B] [C] [D] → Salida     │
│              ↑                         │
│         Primero en entrar = Primero en salir│
└─────────────────────────────────────────┘

Ejemplo de la vida real:
Imagina que tienes un río y construyes un embalse (presa). El agua entra al embalse
y sale poco a poco. El embalse es el buffer - almacena agua (datos) temporalmente
para que no haya inundaciones (sobrecarga) ni sequías (falta de datos).


SU FUNCIÓN PRINCIPAL ES:

1. EVITAR QUE EL PROGRAMA SE QUEDE SIN DATOS (underflow)
   - Cuando los datos llegan más lento de lo que se procesan
   - Ejemplo: Ver un video en streaming con conexión lenta

2. EVITAR SOBRECARGAR EL SISTEMA (overflow)
   - Cuando los datos llegan más rápido de lo que se pueden procesar
   - Ejemplo: Grabar pantalla muy rápido y guardar a disco lento

3. SINCRONIZAR VELOCIDADES DIFERENTES
   - El buffer actúa como "puente" entre dos sistemas que trabajan a velocidades distintas
   - Ejemplo: Cámara graba a 60 FPS pero el disco guarda a 30 FPS


PROBLEMAS COMUNES AL IMPLEMENTAR UN BUFFER

1. DESBORDAMIENTO DE BÚFER (OVERFLOW / OVERRUN)
   
   El problema ocurre cuando el tamaño del búfer es insuficiente para almacenar
   la información que se intenta ingresar.
   
   Generalmente es consecuencia de un error de programación en el que el software
   malinterpreta la capacidad real del espacio reservado.
   
   Ejemplo: Tienes un vaso (buffer) de 250ml y tratas de verter 1 litro de agua.
   El agua se derrama = datos perdidos.
   
   Solución: Verificar siempre el tamaño antes de escribir, o hacer el buffer
   dinámico (que crezca según necesidad).

2. SUBDESBORDAMIENTO DE BÚFER (UNDERFLOW / UNDERRUN)
   
   Es un problema generado por la falta de información en el búfer cuando es
   requerida por el programa que consume los datos.
   
   Ejemplo: Estás reproduciendo un video y se "congela" porque los datos no
   llegaron a tiempo.
   
   Solución: Pre-cargar datos antes de necesitarlos (buffer de-ahead).

3. SOBRESCRITURA (OVERWRITE)
   
   Es un problema generalmente creado por la mala programación del búfer ya que
   se escribe información en espacios de memoria donde todavía no se utiliza,
   causando la pérdida de dicha información.
   
   Ejemplo: Tienes un buffer de 5 espacios. Llegas al espacio 6 y en vez de
   esperar o expandir, sobrescribes el espacio 1 que aún no había sido leído.
   
   Solución: Usar un buffer circular (explicado más adelante).


¿QUE ES UN BUFFER CIRCULAR?

Un buffer circular (también llamado ring buffer o circular queue) es un buffer
de tamaño fijo y finito en el cual hay dos índices:

- Índice A: "Cabeza" (HEAD) - Indica dónde se escribe/nuevos datos entran
- Índice B: "Cola" (TAIL) - Indica dónde se lee/datos salen

Básicamente utilizamos la memoria de forma circular: cuando llegamos al final,
saltamos al inicio. Es como un reloj - después de las 12 vuelve a la 1.

Ejemplo visual de buffer circular:

   [0] [1] [2] [3] [4] [5] [6] [7] [9]
    ↑                           ↑
   HEAD (escribe aquí)         TAIL (lee desde aquí)

   Después de escribir en [9], HEAD vuelve a [0]:
   
   [0] [1] [2] [3] [4] [5] [6] [7] [9]
                     ↑       ↑
                    TAIL    HEAD (escribió aquí)

   Y cuando lees desde TAIL y llegas al final, también vuelve al inicio.


CARACTERÍSTICAS DEL BUFFER CIRCULAR:

1. TAMAÑO FIJO - No crece dinámicamente (evita problemas de memoria)

2. USO EFICIENTE DE MEMORIA - Se reutiliza el mismo espacio siempre

3. SIN COPIAS DE DATOS - No es necesario mover datos, solo mover los índices

4. THREAD-SAFE (si se implementa correctamente) - Permite producer/consumer


¿POR QUÉ USAR BUFFER CIRCULAR PARA CAPTURA DE VIDEO?

En captura de video (como lo que hace AutoClips):

1. EL JUEGO GENERA FRAMES CONSTANTEMENTE
   - Ejemplo: 60 frames por segundo (60 FPS)
   - Cada frame es una imagen (típicamente 1920x1080 = millones de píxeles)

2. NO PODEMOS GUARDAR TODO A DISCO EN TIEMPO REAL
   - Escribir a disco es lento comparada con la velocidad de la RAM
   - Además, no queremos guardar TODO el gameplay, solo los últimos X segundos

3. SOLUCIÓN: BUFFER CIRCULAR EN MEMORIA RAM
   - Guardamos los últimos 30 segundos en memoria (buffer circular)
   - Cuando el buffer se llena, los datos más antiguos se sobrescriben (comportamiento correcto)
   - Solo guardamos a disco cuando el usuario presiona F8 (capturar clip)

Ejemplo práctico:
- Buffer circular de 30 segundos
- Cada segundo = 60 frames
- Total frames en buffer = 30 × 60 = 1800 frames
- Memoria necesaria = 1800 × ~2MB por frame (comprimido) ≈ 3.6 GB

Pero podemos optimizar:
- Usar codec que comprima en tiempo real
- Reducir resolución del buffer si es necesario


FLUJO DE TRABAJO EN AUTOCLIPS:

1. GameWatcher detecta que un juego está corriendo
2. Se captura cada frame del juego
3. Cada frame se escribe en el buffer circular (en HEAD)
4. El buffer mantiene los últimos X segundos (los más antiguos se sobrescriben)
5. Usuario presiona F8 (HotkeyListener)
6. Se toma todo el contenido del buffer (desde TAIL hasta HEAD)
7. Se codifica como video y se guarda en disco(Carpeta destinada)


CÓMO CALCULAR EL TAMAÑO DEL BUFFER:

Fórmula básica:
Buffer tamaño (en frames) = Duración deseada (segundos) × FPS

Ejemplos:
- 30 segundos a 60 FPS = 1800 frames
- 60 segundos a 30 FPS = 1800 frames
- 15 segundos a 60 FPS = 900 frames


PRINCIPIOS DEL PROGRAMA QUE NO SE PUEDEN ROMPER:

1. QUE PUEDA CORRER EN EQUIPOS DE BAJA GAMA
   - No exigir muchos recursos
   - Buffer pequeño pero funcional

2. QUE CUMPLA CON SU FUNCIÓN - SACAR CLIPS
   - Debe capturar correctamente cuando se presiona F8
   - El clip debe tener los segundos JUSTO ANTES de presionar F8

3. QUE NO CONSUMA MUCHO - SER LITE
   - Optimizar memoria
   - Usar compresión eficiente
   - No hacer operaciones innecesarias


IMPLEMENTACIÓN BÁSICA EN C#:

// Buffer circular simple para bytes
public class CircularBuffer
{
    private readonly byte[] buffer;
    private int head = 0;  // Índice de escritura
    private int tail = 0;  // Índice de lectura
    private int count = 0; // Elementos actuales

    public CircularBuffer(int capacity)
    {
        buffer = new byte[capacity];
    }

    // Escribir datos
    public void Write(byte[] data)
    {
        foreach (var b in data)
        {
            buffer[head] = b;
            head = (head + 1) % buffer.Length;
            
            if (count == buffer.Length)
            {
                // Buffer lleno, sobrescribir oldest (comportamiento correcto)
                tail = (tail + 1) % buffer.Length;
            }
            else
            {
                count++;
            }
        }
    }

    // Leer todos los datos disponibles
    public byte[] Read()
    {
        if (count == 0) return Array.Empty<byte>();
        
        var result = new byte[count];
        for (int i = 0; i < count; i++)
        {
            result[i] = buffer[tail];
            tail = (tail + 1) % buffer.Length;
        }
        
        count = 0;
        head = 0;
        tail = 0;
        
        return result;
    }
}
